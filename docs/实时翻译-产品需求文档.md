## 1. 背景与目标

### 1.1 背景

用户需要在 **本地环境** 中，将 **电脑麦克风输入** 经过 **语音识别（ASR）** 转为文本，再调用 **大语言模型（LLM）API** 进行 **实时或准实时翻译**，并在图形界面中展示。ASR 模型体积较大，需支持 **用户主动点击「安装」后从云端下载**；翻译能力依赖 **可配置的第三方 LLM API**。

### 1.2 产品目标

- **G1**：在 macOS 上稳定完成「麦克风 → ASR → 翻译 → 展示」闭环，延迟可接受（准实时）。
- **G2**：ASR 与 LLM 配置 **可本地持久化**；敏感配置 **不经由公网页面泄露**（通过本机后端代理与本地访问策略保障）。
- **G3（第一期核心）**：交付形态为 **Web UI + 本地部署服务** —— 用户在本机启动 **前端（浏览器访问）+ 后端（FastAPI 等）**，通过 `**http://127.0.0.1` / `localhost`** 使用完整能力；**不依赖** 桌面壳即可验收 MVP。
- **G4（第二期）**：在 Web 版稳定后，再评估 **打包为 macOS 可安装应用**（如 Tauri / Electron），统一启动、数据目录与系统权限体验；**不作为第一期交付门槛**。
- **G5**：界面由 **固定侧栏 + 主内容区** 构成：侧栏含 **首页、实时同传、历史记录、收藏短语、设置** 等入口（结构参考 Memo 式效率工具）；**进入应用后的默认主内容区** 为 **落地页**（浅色渐变、居中主行动点）；用户通过 **落地页「开始同传」** 或侧栏 **「实时同传」** 进入 **同传进行中** 视图（双卡片原文/译文区 + 顶栏语言与模式 + 底部操作条，见 §4.1.3）。设置页内仍保留「三方集成」「语言模型」等分区（见 §4.1）。

### 1.3 非目标

- 不提供 WordPress / 纯 CMS 类建站方案。
- **第一期不做** macOS `.app` / `.dmg` 打包、Notarization、应用商店分发（归入第二期）。
- 第一期不承诺「公网多用户 SaaS」部署与账号体系（除非后续单独立项）。
- 第一期不强制支持 Windows / Linux（可作为后续扩展；架构上保留跨平台可能即可）。

## 2. 最优方案选定（已决策）

以下为综合 **开发效率、本地 ASR 生态、先 Web 后桌面、安全与可维护性** 后的 **推荐主方案**。实施顺序上 **优先完成「浏览器 + 本机服务」**；桌面壳仅在第二期引入。

### 2.1 总体架构（第一期：Web UI + 本地服务）


| 层级                     | 选型                               | 说明                                                                                                                                |
| ---------------------- | -------------------------------- | --------------------------------------------------------------------------------------------------------------------------------- |
| **访问方式**               | **本机浏览器 + 本地部署**                 | 前端由 **Vite dev** 或 **构建产物静态服务** 提供；用户通过 `**http://localhost:<前端端口>`** 访问 Web UI；API/WebSocket 指向 **同源或显式配置的 `127.0.0.1:<后端端口>`**。 |
| **前端（Web UI）**         | **React 18 + TypeScript + Vite** | SPA；不做 SSR；与第二期 Tauri 嵌入 **同一套构建产物** 即可复用。                                                                                        |
| **组件与样式**              | **Tailwind CSS + Radix UI**      | **侧栏** 可采用 **深色** 效率工具风格（与 Memo 参考一致）；**主内容区** 采用 **浅蓝紫渐变 + 玻璃拟态（backdrop-blur）+ 大圆角**，对齐「落地页 / 同传页」参考；设置表单与对话框沿用 Radix。          |
| **本地后端（BFF + ASR 编排）** | **Python 3.11+ + FastAPI**       | **ASR 侧优先选 `faster-whisper`（CTranslate2）**；提供 HTTP/WebSocket、LLM 代理、配置与模型安装 API；**仅监听 127.0.0.1**（可配置）。                           |
| **ASR 模型分发**           | **远端 Manifest（JSON）+ 本机下载器**     | 列表来自 manifest；点击「安装」后下载、校验哈希、解压至 **本机约定目录**（如项目数据目录或 `~/.local/share/<app>/asr/`）；支持卸载。                                           |
| **LLM 接入**             | **OpenAI Chat Completions 兼容协议** | Key / Base URL / Model；**保存与测试经后端**；设置页 **保存 / 测试连接**。                                                                            |


### 2.1b 第二期扩展（可选，不阻塞第一期）


| 层级      | 选型                            | 说明                                                                             |
| ------- | ----------------------------- | ------------------------------------------------------------------------------ |
| **桌面壳** | **Tauri 2**（首选）或 **Electron** | 嵌入同一 Web UI；负责一键启停后端、统一应用数据目录、系统麦克风权限提示；**Python 运行时或 sidecar** 在第二期与打包方案一并敲定。 |


**不采用 Next.js 作为主框架的原因：** 第一期无 SSR/SEO 诉求；ASR 与长连接更适合独立 FastAPI 服务。

**不采用 WordPress：** 与实时音频流水线、本地模型管理无关。

### 2.2 关键数据流

1. 用户在 **落地页** 点击 **「开始同传」**，进入 **同传进行中** 视图，开始「实时录制 / 翻译」会话。
2. 前端通过 **WebSocket**（或 Web Audio + 分片上传）将 **音频流/分段音频** 发送至 **FastAPI**。
3. FastAPI 调用 **已安装的本地 ASR 引擎** 输出 **增量识别文本**（原文）。
4. 前端或后端按策略将原文 **分句/分段** 调用 **LLM 翻译接口**（经本地代理，Key 来自本机存储）。
5. 译文以 **流式（SSE 或 chunk）** 返回 UI 展示。

### 2.3 安全与隐私

- **API Key、Base URL**：由 **后端** 持久化至本机（加密文件或受限权限目录）；前端 **不** 把 Key 写入 `localStorage` 明文；LLM 请求 **仅由后端** 代发。**第二期** 可升级为系统钥匙串（Keychain）由桌面壳写入。
- **网络绑定**：后端默认 **仅 `127.0.0.1`**，避免局域网误暴露；文档中说明勿改为 `0.0.0.0` 除非用户明确需要且知悉风险。
- **麦克风**：浏览器 `**localhost` 访问可使用 getUserMedia**（无需 HTTPS）；若未来非 localhost 部署，需 **HTTPS** 才能取麦（第一期不展开）。
- **麦克风音频数据**：默认 **不出网**；ASR 在本地；**仅翻译相关文本** 发往用户配置的 LLM 端点。
- **Manifest 与模型包**：使用 **HTTPS** 下载；**SHA256 校验**；可选签名验证（后续增强）。

### 3.1 首期用户

- **主要用户：** 产品作者本人。
- **使用环境：** macOS，单用户，本机安装。

## 4. 功能需求

### 4.1 布局、导航与界面参考（侧栏 + 双状态主内容区）

整体为 **左侧固定侧栏 + 右侧主内容区**。侧栏负责全局导航与 **设置**；主内容区在 **落地页** 与 **同传进行中** 两态之间切换（路由或同级视图状态均可，需在实现中保持 URL 或可分享状态一致的可选空间）。

#### 4.1.1 侧栏（参考 Memo 式结构 + 「利群翻译」线框）

- **视觉**：建议 **深色背景** + 当前选中项 **浅紫/品牌色高亮**；顶部 **Logo + 产品名**（如「利群翻译 / Liqun Translate」，文案可本地化）；宽度固定，支持折叠（可选，MVP 可不做）。
- **导航项（与参考页对齐；分期见各 FR）**：
  - **首页**：主内容区为 **落地页**（§4.1.2）。
  - **实时同传**：进入 **同传进行中**（§4.1.3）；若 ASR/LLM 未就绪，**拦截并引导** 至设置（同 FR-ASR-06 策略）。
  - **历史记录**：会话/转写历史列表与回看（**首期可简化为占位或仅本地最近 N 条**，见 FR-HIST-01）。
  - **收藏短语**：常用句收藏与插入（**首期可占位**，见 FR-FAV-01）。
  - **设置**：**语言模型**、**三方集成（ASR）**、**转写设置**、**翻译设置**、**关于** 等（与既有 FR 一致）。
- **可扩展占位**：其它模块 **首期可隐藏** 或标注 Coming soon。

#### 4.1.2 落地页（进入应用后的默认视图；参考用户提供「星火同传」式落地页）

主内容区 **居中构图**，浅蓝紫渐变背景、轻量阴影与大圆角，强调 **单次主行动点**。

- **顶栏（主内容区顶部，可与全局顶栏合并）**：
  - **左侧**：产品 Logo + 名称（文案可本地化，非必须与参考图一致）。
  - **语言方向**：**源语言 → 目标语言** 下拉/选择器（如「英 → 中」），与同传会话共用同一套选择状态。
  - **右侧（MVP 策略）**：
    - **语音播报**：开关 + 图标；若 **TTS 未实现**，可为 **占位** 或灰显并提示「即将推出」。
    - **二维码等**：首期 **可省略** 或占位。
    - **「下载客户端」**：本地 Web 阶段建议改为 **「桌面版敬请期待」** 或 **隐藏**，避免误导；第二期再对接真实下载。
- **中部**：
  - **主标题 + 副标题**（产品名 + 一句价值说明）。
  - **视觉焦点**：插画/3D 形象 **可用占位图**，后续替换；旁侧可有 **示例双语气泡**（静态演示即可，MVP 可不接真实音频）。
  - **主按钮**：**「开始同传」**（或「开始实时翻译」），点击后 **进入 §4.1.3**；若 ASR 未安装或 LLM 未配置，**拦截并引导** 至设置（符合 FR-ASR-06 等）。
- **底部**：**免责声明 / 模型说明** 小字（本地部署可写「内容由本地 ASR 与用户配置的 LLM 生成」等）。
- **配额文案**（如「本月剩余次数」）：**本地自用版可不展示**；若展示需说明为占位或与后端策略挂钩。

#### 4.1.3 同传进行中（「开始同传」或侧栏「实时同传」；参考「利群翻译」同传页）

由 **落地页主按钮** 或 **侧栏「实时同传」** 进入；与落地页 **共用源/目标语言状态**；整体为 **浅主区 + 白卡片 + 紫色主 CTA**，可与落地页同一套渐变背景。

**主内容区顶栏（会话内）**

- **语言方向（居中）**：**源语言**、**目标语言** 两个下拉；中间 **交换箭头** 一键对调源/目标（与落地页选择器逻辑一致）。
- **模式（右侧）**：**「实时模式」**（或等价文案）带 **声波图标** + **下拉**，用于切换 **处理模式**（如：实时分段 / 句级稳定 等；具体枚举在「翻译设置」或此处二级菜单中配置，MVP 可仅一种模式 + 下拉占位）。
- **帮助**：右上角 **问号** 入口 → 简短说明、快捷键或链至「关于」（MVP 可弹窗 Markdown）。

**主展示区：上下双卡片（非单气泡叠层）**

- **原文卡片（上）**
  - **标题**：`原文（{源语言名}）`，与当前所选语言一致。
  - **状态条**：**音频波形**（示意正在拾音/处理）+ **会话计时**（如 `00:00:18`，从进入同传页或开始监听起算，规则在实现中统一）。
  - **工具**：卡片右上角 **朗读（TTS）**、**复制**（复制当前卡片展示文本或全文策略见 FR-TR-08）。
  - **正文区**：**主行** 为 ASR **当前段/累计段** 大字号展示；**次行** 为 **浅灰小字「预览译文」**（与主译文区同步或略超前/略滞后，用于扫读；可与主译文重复或仅展示当前句预览，见 FR-TR-07）。
- **译文卡片（下）**
  - **标题**：`译文（{目标语言名}）`。
  - **工具**：右上角 **朗读（TTS）**、**复制**。
  - **正文区**：译文 **大字号**；**右下角状态**：绿点 + **「自动翻译中」**（或空闲/暂停/错误等态），与全局会话状态联动。

**长文本与性能**：随会话进行，双卡片内可 **追加段落或虚拟滚动**；需避免一次渲染过长 DOM（实现选型由研发定）。

**底部操作条**

- **主 CTA（左侧，实心品牌紫、大按钮）**：与 **同传流水线状态** 绑定，**不得** 与「未进入同传页」的落地页主按钮共用同一套文案逻辑。详见下节 **「主按钮与状态」**。
- **导出文本**（中）：导出 **本会话原文+译文**（格式：TXT/Markdown 等至少一种，见 FR-TR-09）。
- **语音设置**（右，下拉）：**输入/输出设备、系统音量相关选项** 的聚合入口（浏览器能力内枚举麦克风；扬声器/ TTS 声音可选，见 FR-TR-10）。

**主按钮与状态（进入同传页后）**

用户 **进入同传进行中页**（§4.1.3）且 **成功进入可监听状态** 后，底部主按钮 **默认** 显示 **「暂停同传」**，表示当前 **拾音与翻译流水线处于进行中**。点击后在 **「暂停同传」** 与 **「继续同传」** 之间切换，并驱动下列 **会话子状态**（实现须与前端展示、后端/WebSocket 一致）：


| 状态 ID（实现用） | 用户理解      | 主按钮文案                       | 拾音 / ASR 送流  | 新译文请求 | 会话计时（见 FR-TR-12） | 波形示意  | 译文卡片角标（示例）     |
| ---------- | --------- | --------------------------- | ------------ | ----- | ---------------- | ----- | -------------- |
| **LIVE**   | 进行中 / 监听中 | **暂停同传**                    | 开            | 允许    | **累计走动**         | 活跃    | 绿点 + **自动翻译中** |
| **PAUSED** | 已暂停       | **继续同传**                    | 关（或缓冲不入 ASR） | 暂停新发  | **暂停累计**         | 静止或隐藏 | 灰/黄点 + **已暂停** |
| **ARMING** | 准备中（可选）   | 「准备中…」或禁用                   | 未开           | 否     | 未开始或 00:00       | 未起    | **连接中** / 无    |
| **ERROR**  | 异常需恢复     | **重试** 或 **继续**（见 FR-TR-04） | 关            | 暂停    | 暂停累计             | 静止    | 红点 + **出错：…**  |


**约定（MVP）**

1. **默认路径**：进入同传页且麦克风等前置条件满足后，应 **尽快** 进入 **LIVE**，主按钮为 **「暂停同传」**；若需用户二次确认开麦，**ARMING** 可简化为按钮禁用 + 顶栏提示，不必单独页面。
2. **LIVE → PAUSED**：用户点 **「暂停同传」** → 进入 **PAUSED**，按钮文案变为 **「继续同传」**；**停止向 ASR 送新音频**（或清空缓冲策略在技术方案中写明），**不再发起新的翻译请求**；**计时器暂停**；波形静止。
3. **PAUSED → LIVE**：用户点 **「继续同传」** → 回到 **LIVE**，按钮恢复 **「暂停同传」**；恢复拾音与翻译；**计时器从冻结读数继续累计**（**暂停期间读数不变**，与 FR-TR-12 一致；即展示为「累计 LIVE 时长」，非整页停留墙钟）。
4. **ERROR**：从 LIVE/PAUSED 因麦克风掉线、ASR/LLM 失败等进入；主按钮优先 **「重试」**（回到 LIVE 或重新初始化）；若错误与「仅翻译失败」相关，可允许 **「继续同传」** 仅恢复拾音并在译文区提示部分失败（细节见 FR-TR-04）。
5. **与落地页区别**：落地页主按钮为 **「开始同传」**；**仅在同传页内** 使用 **暂停/继续** 文案，避免同一按钮在三处文案混淆。

**与旧版「单区玻璃气泡」关系**：若实现沿用玻璃拟态，**以本双卡片信息架构为准**；视觉可保留 blur/大圆角，但 **信息分区** 应对齐本条。

#### 4.1.4 设计令牌（建议）

- **主内容区背景**：浅蓝 — 淡紫 **线性渐变**；卡片 **白/半透 + blur**。
- **主 CTA 按钮**：实心品牌色（可与参考图蓝/紫接近，具体色值在 UI 规范中锁定）。
- **侧栏与主内容区对比**：侧栏深、主区浅，形成清晰 **「导航 / 任务」** 分层。

### 4.2 三方集成 — ASR 模型（参考「安装才下载」）

- **FR-ASR-01**：展示 ASR **可用列表**（名称、版本、简介、图标），数据来自 **远端 manifest**。
- **FR-ASR-02**：根据本机状态显示 **未安装 →「安装」**；**已安装 →「卸载」**（或删除图标），与参考交互一致。
- **FR-ASR-03**：点击「安装」后 **从云端拉取** 对应包（支持 **进度**、**暂停/重试**、失败原因提示）。
- **FR-ASR-04**：安装完成后 **校验哈希**；写入本机 **安装清单**（id、版本、路径、时间）。
- **FR-ASR-05**：卸载时删除本地文件并更新清单；若当前会话占用该资源，需 **阻止或二次确认**。
- **FR-ASR-06**：会话开始前检测 **已选 ASR 是否已安装**；未安装则引导至集成页。

### 4.3 语言模型 — LLM API（参考多厂商表单）

- **FR-LLM-01**：左侧 **厂商列表**（首期至少：OpenAI 兼容占位 + **自定义 Base URL**；可配置展示 OpenAI、DeepSeek、智谱、Moonshot、Ollama 等 **同一表单结构**）。
- **FR-LLM-02**：表单字段：**API Key（密文）**、**Base URL**、**模型（下拉或手输）**；必填项标星；支持 **显示/隐藏 Key**。
- **FR-LLM-03**：**保存** 写入本机配置；**测试** 发起最小请求并展示成功/失败信息（不打印完整 Key）。
- **FR-LLM-04**：支持 **多配置档案**（可选 MVP：仅单档案；Phase 1.1 可多档案 + 默认项）。

### 4.4 翻译与会话

- **FR-TR-00**：**页面状态机** — **落地页**（默认）→ **「开始同传」或侧栏「实时同传」** → **同传进行中**（内部在 **LIVE / PAUSED / ERROR** 间切换，见 §4.1.3）→ **结束会话**（显式按钮或返回首页）后回到落地页或进入历史；PRD 要求 **至少** 能完成「再开一轮」。**侧栏切换** 离开同传页时，须 **暂停或结束会话** 的策略在交互稿中统一（建议：二次确认或自动切 **PAUSED**）。
- **FR-TR-01**：用户可选择 **源语言 / 目标语言**（首期可「自动检测 + 目标语」）；**落地页顶栏 / 同传页顶栏 / 卡片标题** 展示的语言名 **一致**；**交换按钮** 对调源/目标并 **刷新会话内展示标签**。
- **FR-TR-02**：**同传进行中** 按 §4.1.3 展示 **双卡片：原文区 + 译文区**；流水线子状态至少包含 **LIVE（进行中）**、**PAUSED（已暂停）**、**ERROR（异常）**，可选 **ARMING（准备中）**；状态与 §4.1.3 表格一致。**译文卡片右下角** 以文案+色点反馈 **「自动翻译中」/「已暂停」/ 错误摘要** 等，须与主按钮状态 **一致**。
- **FR-TR-03**：翻译策略（MVP）：**按句或按时间片聚合** 后调用 LLM，平衡延迟与质量；**顶栏「实时模式」** 若提供多模式，仅切换 **聚合/流式策略** 的预设，具体参数可在「翻译设置」同步暴露。
- **FR-TR-04**：错误处理：麦克风拒绝、ASR 崩溃、LLM 429/5xx — **可恢复提示** 与重试入口；错误时状态点与 **「自动翻译中」** 区分（如红/黄点 + 文案）。
- **FR-TR-05**：**同传页** 须具备 **主 CTA**，且在 **LIVE** 时显示 **「暂停同传」**，在 **PAUSED** 时显示 **「继续同传」**（见 §4.1.3 主按钮与状态表）；须具备 **会话计时**、**波形或等价监听示意**；**PAUSED** 时停止送麦并停止新发翻译请求；前后端/WebSocket 对 LIVE/PAUSED **同步**。
- **FR-TR-12**：**会话计时**：从进入 **LIVE** 首次开始或从用户首次点「继续同传」起算（与 §4.1.3 约定一致）；**PAUSED 期间计时暂停**；**ERROR** 期间默认暂停；重新 **LIVE** 后继续累计。是否在 UI 旁注「有效时长」为可选增强。
- **FR-TR-06**：**帮助**：同传页 **问号** 入口可打开帮助（短文案或链至文档）；不阻塞同传。
- **FR-TR-07**：**原文卡片内预览译文**：在 ASR 主文本下方以 **次级样式** 展示当前段对应译文（可与译文卡片内容同源，避免两套逻辑长期不一致）。
- **FR-TR-08**：**复制 / 朗读**：原文卡、译文卡各自 **复制**；**TTS** 若首期未实现，按钮 **灰显 + 提示**，占位需可测。
- **FR-TR-09**：**导出文本**：导出当前会话 **原文与译文**（至少一种可读格式）；文件名含时间戳；**不** 在导出文件中写入 API Key。
- **FR-TR-10**：**语音设置**：在同传页可 **选择输入设备**（浏览器 `enumerateDevices` 能力内）；输出相关选项与系统/TTS 策略一致；与落地页设备选择 **可共享状态**（可选）。

### 4.4b 历史与收藏（与参考导航对齐；可分阶段）

- **FR-HIST-01**：**历史记录**：保存 **会话级** 元数据（起止时间、语言对、摘要）与 **可回看的原文/译文**（本地存储或后端文件，单用户）；MVP 可 **仅列表 + 详情**；**导出** 可与 FR-TR-09 复用逻辑。
- **FR-FAV-01**：**收藏短语**：用户可 **收藏** 句段并在新会话中 **插入**（首期可 **占位页**「即将推出」，不阻塞 M5）。

### 4.5 设置与其它

- **FR-ST-01**：**转写设置**：默认 ASR 模型、VAD/分段参数（高阶可后期）。
- **FR-ST-02**：**关于**：版本号、开源许可、数据说明（哪些数据出网）。

## 7. 风险与依赖

- **ASR 实时性 vs 硬件：** 小模型快但准度低；大模型准但慢 —— 需在 manifest 中提供 **推荐档位**（如 small / medium）。
- **LLM 流式与费用：** 高频小请求可能增加费用；需 **合并窗口** 与 **限流** 策略。
- **第一期：** 用户需本机已装 **Python 3.11+**（或文档提供 `uv`/conda 指引）；**不** 在第一期阻塞于「单文件无 Python 安装包」。
- **第二期：** Python 运行时与后端 **捆绑方式**（PyInstaller sidecar、嵌入式解释器等）需在 **M6 前做 Spike**，与 Tauri 启动链路一并验证。
